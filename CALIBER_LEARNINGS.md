# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[gotcha]** The `src/` repositories are written against the **v4.x** `league/oauth2-server` API (`League\OAuth2\Server\Storage\*Interface`, `League\OAuth2\Server\Entity\*`). Running `composer install` may resolve to v8.x (which uses `League\OAuth2\Server\Repositories\*RepositoryInterface` and `League\OAuth2\Server\Entities\*`), which will break all imports. The correct version constraint is `^4.1` — verify with `composer show league/oauth2-server | grep version` after install.
- **[fix]** When `composer require` fails with "the package is fixed to X (lock file version) by a partial update and that version does not match", add `-W` (`--with-all-dependencies`) to let Composer upgrade/downgrade locked transitive packages: `composer require --dev "phpunit/phpunit:^9.5" -W --no-interaction`.
- **[gotcha]** PHPUnit 5.7.x is incompatible with PHP 8.3 — it crashes with `Cannot acquire reference to $GLOBALS` in `phpunit/src/Util/Configuration.php`. Use PHPUnit 9.x (`^9.5`) for PHP 8.3+ compatibility. Always use `./vendor/bin/phpunit` (the project-local binary), not the global `phpunit`.
- **[pattern]** The `TestDb` bootstrap shim must define `MYSQLI_ASSOC` (= 1) and `MYSQL_ASSOC` constants before any test runs — the repositories pass `MYSQLI_ASSOC` to `next_record()` but PDO/SQLite environments do not define it.
- **[pattern]** When loading `sql/oauth2.sql` into SQLite for tests, strip these MySQL-specific constructs via `preg_replace`: `ENGINE=\w+`, `DEFAULT CHARSET=\w+`, `COLLATE=[\w_]+`, `COMMENT='...'`, and convert `int(...) unsigned NOT NULL AUTO_INCREMENT` columns to `INTEGER PRIMARY KEY AUTOINCREMENT`. Also drop the now-redundant `PRIMARY KEY (\`id\`)` line that follows and any `FOREIGN KEY ... REFERENCES accounts(...)` constraints referencing the parent project's tables.
- **[pattern]** `TestDb` must extend `MyDb\Generic` and override these methods to wrap a real PDO connection: `query()` (materialise result set into `$this->rows`), `qr()` (return all rows as array), `num_rows()` (count `$rows`), `next_record()` (advance cursor, populate `$this->Record`), `real_escape()` (use `PDO::quote()` then strip the wrapping single-quotes), plus `exec()`, `prepare()`, `getAttribute()`, `setAttribute()`, and `lastInsertId()` as PDO pass-throughs.
- **[convention]** In league/oauth2-server v4.x: storage classes extend `League\OAuth2\Server\Storage\AbstractStorage` (not `AbstractRepository`); interfaces live in `League\OAuth2\Server\Storage\*Interface`; entities live in `League\OAuth2\Server\Entity\*Entity`. The `$this->server` reference is set by `AbstractStorage::setServer()` — use `$this->server` (not `$this->getServer()`) for direct entity instantiation like `new AccessTokenEntity($this->server)`.
- **[convention]** `ClientEntity::hydrate()` in v4.x accepts a flat associative array — use it instead of individual setters: `$client->hydrate(['id' => ..., 'name' => ...])`. For `ScopeEntity`, `hydrate(['id' => ..., 'description' => ...])` works the same way.

- **[pattern]** Hardcode the SQLite test schema directly as a heredoc in `MyDbTest` rather than loading and transforming `sql/oauth2.sql` at runtime. Runtime regex transformation of MySQL DDL is fragile (KEY clauses, inline comments, unsigned types) and breaks silently. The hardcoded schema in `tests/bootstrap.php` is the authoritative test schema.
- **[gotcha]** Do NOT enable `PRAGMA foreign_keys = ON` in the SQLite `TestDb`. MySQL (production) does not enforce FK insertion ordering the same way SQLite does, so tests that seed fixtures in column-natural order (e.g. inserting into `oauth_access_tokens` before `oauth_sessions`) will fail with FK violations in SQLite but work fine in MySQL. Leave FK enforcement off so test fixture seeding can follow logical rather than dependency order.
- **[gotcha]** SQLite returns INTEGER column values as PHP `int`, not `string`. Test assertions using `assertSame()` will fail if the SQLite schema declares numeric columns (e.g. `session_id`, `expire_time`) as `INTEGER` and the test expects string values. Declare those columns as `VARCHAR(255)` in the SQLite test schema to match the string values that MySQL/string-escaped INSERT queries produce.
- **[gotcha]** `RefreshTokenEntity` in league/oauth2-server v4.x has no public `getAccessTokenId()` method — the access token ID is stored in the private `$accessTokenId` property and is only accessible via `getAccessToken()` (which requires a fully-wired server with an access token storage). In tests, read it via `ReflectionProperty`: `$ref = new ReflectionProperty($entity, 'accessTokenId'); $ref->setAccessible(true); $this->assertEquals('id', $ref->getValue($entity));`
- **[gotcha]** `$this->db->real_escape()` coerces `null` to an empty string `''`, wrapping it in quotes and producing `""` in the SQL. This causes NOT NULL constraints to silently pass (empty string is not null). Use a `sqlValue($value)` helper on `Repository` that returns the literal string `'NULL'` for PHP `null` and `'"'.$this->db->real_escape($value).'"'` otherwise — so database NOT NULL and FK constraints actually fire during tests.
- **[fix]** `phpunit.xml` using PHPUnit 5.x schema (no `xsi:noNamespaceSchemaLocation`, `syntaxCheck="true"` attribute, `<testsuite>` missing `name`, `<filter><whitelist>`) causes PHPUnit 9 to warn and may produce incorrect test results. Update to PHPUnit 9 schema: add `xmlns:xsi` + `xsi:noNamespaceSchemaLocation`, remove `syntaxCheck`, add `name` to `<testsuite>`, replace `<filter><whitelist>` with `<coverage><include>`.
- **[fix]** `sed` fails silently (or with "unknown option to s") when the replacement pattern contains unescaped `|` characters. For multi-file regex substitutions involving alternation patterns, use Python3 `re.sub()` instead: `python3 -c "import re,os; ..."` — it handles complex patterns reliably without shell escaping issues.
